# Config Schema

<Badge variant="secondary">Configuration</Badge>

Source of truth: `taki-cli/README.md` plus runtime validation in CLI core.

## Minimal valid config

```json
{
  "services": [
    {
      "name": "web",
      "command": "npm",
      "args": ["run", "dev"],
      "color": "green"
    }
  ],
  "maxLogLines": 200,
  "ui": {
    "theme": "vscode-dark-plus"
  }
}
```

## Health checks

<Tabs defaultValue="log" className="my-6">
  <TabsList>
    <TabsTrigger value="log">Log pattern</TabsTrigger>
    <TabsTrigger value="http">HTTP endpoint</TabsTrigger>
  </TabsList>

  <TabsContent value="log">

```json
{
  "type": "log",
  "pattern": "ready",
  "timeoutMs": 20000
}
```

Use this when your service prints a reliable startup message.

  </TabsContent>

  <TabsContent value="http">

```json
{
  "type": "http",
  "url": "http://127.0.0.1:3000/health",
  "intervalMs": 500,
  "timeoutMs": 20000
}
```

Use this for APIs or web services with explicit health endpoints.

  </TabsContent>
</Tabs>

## Common patterns

```json
{
  "services": [
    {
      "name": "web",
      "command": "npm",
      "args": ["run", "dev"],
      "startAfter": ["api"]
    },
    {
      "name": "api",
      "command": "npm",
      "args": ["run", "dev:api"],
      "healthCheck": {
        "type": "http",
        "url": "http://127.0.0.1:3001/health",
        "intervalMs": 1000,
        "timeoutMs": 30000
      }
    }
  ]
}
```
